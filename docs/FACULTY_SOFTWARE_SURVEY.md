## How to use this document

1. Print or share this with each department's lab in-charge
2. For each row, the faculty marks: **Need** / **Nice to have** / **Don't use**
3. If "Need" — we check if the open-source alternative is acceptable
4. If the open-source alternative is NOT acceptable — that software goes on the purchase list
5. If "Don't use" — we skip it entirely
