# Live Patch Log — 2026-04

This file records every ad-hoc SSH patch applied directly to a live
host outside normal git deployment flow.

Why this exists:
- live SSH edits create drift that git history does not capture
- they can be erased by the next file copy, pull, or host rebuild
- the rig needs one place that says what changed, why, and whether a
  canonical repo-side fix exists yet

## Rules

1. Add a row within 5 minutes of any live SSH patch.
2. Record the exact live host, file or DB touched, and the operator
   reason.
3. If a repo commit later supersedes the patch, fill the
   `canonical_followup` column with that commit hash and subject.
4. If no repo fix exists yet, write `NONE YET`.
5. Do not treat this file as approval to keep patching live. It is an
   audit trail, not a deployment strategy.
