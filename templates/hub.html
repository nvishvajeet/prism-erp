Three-Engine Development Model
Engine 1: LLM (Claude) — reads code, designs, writes patches, orchestrates both machines.
Engine 2: MacBook Pro — local tests, crawlers, smoke_test.py.
Engine 3: Mac Mini — SSH remote verification, production deploy, parallel compute.

Pattern: Fire local + mini verification jobs in background simultaneously, then LLM works while machines grind. Zero idle compute. Every verification task that a machine can do saves ~3M tokens of LLM reasoning per session.

SSH: Single unified ED25519 key at ~/.ssh/id_ed25519. Alias: prism-mini in ~/.ssh/config. Works for GitHub, Bitbucket, GitLab, and the mini.
AI Agent Workflow Rules (12 rules)
  1. Read what you need, not everything. CLAUDE.md auto-loads. Read AGENT_POLICY.md and git-server/README.md only when you need a specific recipe.
  2. Pull → work → test → commit → push. No local-only commits. Every push triggers the mirror hook.
  3. Run pre-commit gate before every commit. PRISM: smoke_test.py (~5s mandatory). Other projects: whatever their WORKFLOW.md specifies.
  4. Fix root causes, verify state before acting.
  5. Commit frequently but not excessively. One commit per meaningful unit of work — a unit a future reader could reference, revert, or cherry-pick.
  6. Batch terminal permissions, front-load shell operations.
  7. Full gates only at release boundaries. Mid-flight gate is smoke/sanity only.
  8. Read project's reusable abstractions before adding code.
  9. Hard attributes locked. Changes to hard attributes require major version bump.
  10. Website stays up. Atomic deploys. Never interrupt live users, never force-push stable branch.
  11. Log rare, git-invisible actions with ai-log. Credential changes, infra changes outside git, cross-project actions.
  12. Cross-project auto-tasks — one check per local day, not per session.
Git Hard Rules
  • NEVER open a bare repo in Sourcetree or any GUI client.
  • ONE bare per project, exactly one, under ~/.claude/git-server/.
  • Clone from Level 1 LOCAL bare, never from Level 2 upstream.
  • Bares MUST NOT be in Dropbox, iCloud, or any synced folder.
  • Push to origin (Level 1) always. Level 2 push requires a concrete reason.
  • Never force-push or rewrite history on a release branch.
  • Imperative commit subject ≤ 70 chars, body explains why not what.
Onboard a New Project
Requires explicit user approval. The registry is the gate.

~/.claude/git-server/bin/new-project.sh <name> [existing-working-copy]

This creates the Level-1 bare, installs the post-receive hook, resets origin, and seeds a WORKFLOW.md stub. For Level-2 (mini mirror), add --mirror-to-mini flag.

After onboarding, update the registry in both AGENT_POLICY.md §1 and git-server/README.md §3.
